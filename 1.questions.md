This is a comprehensive roadmap designed to take you from understanding concepts to implementing complex systems.

To maximize your efficiency, **do not** solve these questions linearly from 1 to 30. Instead, I have reorganized them by **Design Pattern clusters**. This allows you to master a specific pattern (like Strategy or Observer) and progressively increase difficulty within that context.

### **The Master Process: How to Solve Every Question**

For every question listed below, follow this strict 5-step loop. Do not write code until Step 3.

1. **Clarify & Scope (5 mins):**
* Define Actors (Admin, User, System).
* Define Use Cases (e.g., "User scans ticket," "System calculates fee").


2. **Diagramming (10 mins):**
* Draw the **UML Class Diagram** (Classes, Interfaces, Relationships).
* Identify where the design pattern fits (e.g., *"I will use Strategy here for Pricing"*).


3. **Skeleton Coding (10 mins):**
* Write the Interfaces and Abstract Classes first.
* Define the public API methods.


4. **Implementation (20 mins):**
* Flesh out the concrete classes.
* Handle thread safety (if applicable).


5. **Review (5 mins):**
* Ask: "If I add a new requirement (e.g., a new vehicle type), do I have to modify existing code?" (Open/Closed Principle).
